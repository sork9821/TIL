### Context

첫 n8n 워크플로우 자동화를 성공적으로 구축했습니다. 이 자동화는 디스코드에서 메시지를 수신하는 것으로 시작하여, 수신된 메시지를 기반으로 agent를 실행합니다. agent는 받은 메시지를 TIL(Today I Learned) 형식의 기술 문서로 변환하고, 이 문서를 GitHub 저장소에 자동으로 push하는 역할을 합니다. 마지막으로, push 작업이 완료되면 README.md 파일에 해당 문서의 인덱스를 추가하고, 디스코드 채널에 완료 메시지를 발송하여 사용자에게 알립니다.

### Core

*   **Trigger**: Discord 메시지 수신
*   **Agent Execution**: 수신된 메시지를 TIL 형식으로 변환
*   **GitHub Push**: 변환된 TIL 문서를 GitHub 저장소에 push
*   **Index Update**: README.md 파일에 TIL 문서 인덱스 추가
*   **Completion Notification**: Discord 채널에 완료 메시지 발송

*(참고: n8n 워크플로우의 상세 노드 설정은 사용자의 환경에 따라 달라질 수 있습니다. 핵심은 각 노드의 input과 output을 명확히 이해하는 것입니다.)*

### Insight

n8n을 처음 사용해보았지만, 워크플로우 자동화 구축이 생각보다 어렵지 않다는 것을 알게 되었습니다. 성공의 핵심은 각 노드가 어떤 입력을 받고 어떤 출력을 생성하는지 명확하게 이해하는 것입니다. 이 두 가지만 정확히 파악하면 복잡해 보이는 워크플로우도 쉽게 설계하고 구현할 수 있습니다. 특히 n8n의 노드 간 데이터 흐름을 시각적으로 이해하는 것이 중요했습니다. n8n 학습에 큰 도움을 준 후츠릿에게 감사합니다.
